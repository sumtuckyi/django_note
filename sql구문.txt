ALTER문
기존의 테이블을 유지하면서 원하는 부분만 변경할 때 사용하는 명령문


제약조건 설정하기
-> CONSTRAINT 제약조건명 제약조건(CHECK, UNIQUE, PRIMARY KEY, FOREIGN KEY(R))
ex) CONSTRAINT 제약조건명 FOREIGN KEY 자식 컬럼명 REPERENCES 부모 컬럼명
외래키를 이용한 부모-자식 상속 테이블의 관계
ON DELETE CASCADE : 부모테이블 참조컬럼에 있는 행 삭제 시, 자식테이블의 참조행도 같이 삭제
ON DELETE SET NULL : 부모테이블 참조컬럼에 있는 행 삭제 시, 자식테이블 참조행의 컬럼값을 NULL로 변경

SEQUENCE
: 정수값을 저장하여 자동 증가하는 값을 제공하는 객체
중복되지 않는 기본기 컬럼에 사용할 값을 발생시키는데 주로 사용한다. 

INDEX
: 테이블에서 저장된 행을 보다 빠르게 검색하기 위한 기능을 제공하는 객체
컬럼에 인덱스를 설정하면 인덱스 영역이 생성되어 컬럼에 대한 행검색 속도 증가
인덱스를 사용하면 전체 테이블을 스캔하지 않고 데이터를 찾을 수 있다. 
조건식에서 많이 사용하는 컬럼에 설정하며, 행이 많을 경우 인덱스를 사용하면 효율적이다.
UNIQUE INDEX: 컬럼에 PK 제약조건이나 UNIQUE 제약조건에 의해 자동으로 생성되는 인덱스
NON-UNIQUE INDEX: 컬럼에 사용자가 수동으로 설정하는 인덱스 

SYNONYM
: 오라클 객체에 별명을 부여하여 사용하는 객체
SCHEMA
: 테이블, 뷰, 시퀀스 등과 같은 객체의 모음
데이터베이스의 구조와 제약조건에 대한 전반적인 명세를 기술한 것으로
외부, 내부, 개념스키마와 같이 3개의 계층으로 나눌 수 있다.
일반적으로 스키마라고 하면 내부스키마(데이터베이스의 물리적 구조)를 가리킨다.
